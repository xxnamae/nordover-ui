Buttons (.btn)

Base button component with variants and sizes.

Base Class & Modifiers

Class Purpose Required? Compatible With Example
.btn Base button styles (padding, border-radius, cursor) Yes All modifiers <button class="btn">
.btn-primary Primary variant (filled, accent color) Choose one .btn-sm, .btn-lg, .btn-elevated, :disabled <button class="btn btn-primary">
.btn-secondary Secondary variant (outlined, subtle) Choose one .btn-sm, .btn-lg, .btn-elevated, :disabled <button class="btn btn-secondary">
.btn-ghost Minimal variant (transparent background) Choose one .btn-sm, .btn-lg, :disabled <button class="btn btn-ghost">
.btn-link Text link variant (no background, underline) Choose one .btn-sm, .btn-lg (size ignored), :disabled <a class="btn btn-link" href="#">
.btn-sm Small size (reduced padding) Optional All variants <button class="btn btn-primary btn-sm">
.btn-lg Large size (increased padding) Optional All variants <button class="btn btn-primary btn-lg">
.btn-elevated Add shadow elevation (tactile feel) Optional .btn-primary, .btn-secondary <button class="btn btn-primary btn-elevated">
:disabled Disabled state (reduced opacity, cursor not-allowed) Optional All variants and sizes <button class="btn btn-primary" disabled>

Valid Combinations

.btn .btn-primary
.btn .btn-primary .btn-sm
.btn .btn-secondary .btn-lg .btn-elevated
.btn .btn-ghost

Invalid Combinations (Avoid)

  • .btn-primary .btn-secondary — Don't mix variants (only use one)
  • .btn-sm .btn-lg — Don't combine sizes (only use one)
  • .btn-link .btn-elevated — Link buttons don't support elevation
  • .btn-ghost .btn-elevated — Ghost buttons don't support elevation

.form-textarea Textarea styling Yes (for textarea) :disabled, :focus, :hover <textarea class="form-textarea">

Checkboxes & Radios

Class Purpose Required? Compatible With Example
.form-checkbox Checkbox input styling Yes (for checkboxes) :checked, :disabled, :focus <input type="checkbox" class="form-checkbox">

Badges

Small status/tag components with color variants.

Base Class & Color Variants

Class Purpose Required? Compatible With Example
.badge Base badge styles (neutral gray) Yes All color variants (optional) <span class="badge">Default</span>
.badge-success Success state (green) Optional .badge <span class="badge badge-success">Done</span>
.badge-error Error state (red) Optional .badge <span class="badge badge-error">Error</span>
.badge-warning Warning state (yellow/orange) Optional .badge <span class="badge badge-warning">Caution</span>
.badge-info Info state (blue/accent) Optional .badge <span class="badge badge-info">Info</span>

Valid Combinations

Neutral Success Error Warning Info

Invalid Combinations (Avoid)

  • .badge-success .badge-error — Don't combine color variants (only use one)
  • Using variants without base .badge class (always include base)

Tables (.data-table, .table)

Data display tables with striping and hover states.

Base Class

Class Purpose Required? Compatible With Example
.data-table or .table Base table styling (border-collapse, padding) Yes (on table element) Standard <thead>, <tbody>, <tr>, <td>, <th> <table class="data-table">

Recommended Pattern

<table class="data-table">
  <thead>
    <tr>
      <th>Name</th>
      <th>Email</th>
      <th>Status</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>John Doe</td>
      <td>john@example.com</td>
      <td><span class="badge badge-success">Active</span></td>
    </tr>
  </tbody>
</table>

Features

  • Automatic hover background on rows
  • Proper padding and alignment
  • Border styling with color tokens
  • Works with responsive utilities (max-width, overflow)

Pagination

Page navigation controls.

Base Classes

Class Purpose Required? Compatible With Example
.pagination Container (flex layout, gap) Yes (on wrapper) .pagination-btn, .pagination-item, .pagination-dot <div class="pagination">
.pagination-btn Previous/Next navigation buttons Optional .pagination, :hover, :disabled <button class="pagination-btn">Previous</button>
.pagination-item Number buttons (1, 2, 3, etc) Optional .pagination, [aria-current="page"] <a class="pagination-item" aria-current="page">1</a>

  <button class="pagination-btn">Previous</button>
  <a class="pagination-item" aria-current="page">1</a>
  <a class="pagination-item">2</a>
  <a class="pagination-item">3</a>
  <button class="pagination-btn">Next</button>

</div>

Best Practices & Patterns

Common patterns and recommendations for component usage.

Form Pattern (Best Practice)

<form class="stack">
  <div class="field">
    <label class="field-label" for="email">Email</label>
    <input type="email" id="email" class="form-input" required>
    <p class="field-help">We'll never share your email.</p>
  </div>

  <div class="field">
    <label class="field-label">Preferences</label>
    <div class="form-group">
      <div class="form-group-item">
        <input type="checkbox" id="updates" class="form-checkbox">
        <label for="updates">Receive updates</label>
      </div>
    </div>
  </div>

  <div class="cluster">
    <button type="submit" class="btn btn-primary">Submit</button>
    <button type="reset" class="btn btn-secondary">Clear</button>
  </div>
</form>

Action Bar Pattern (Best Practice)

<div class="cluster">
  <div>
    <h3>Results</h3>
  </div>
  <div class="cluster" style="margin-left: auto;">
    <button class="btn btn-secondary btn-sm">Filter</button>
    <button class="btn btn-primary btn-sm">Add New</button>
  </div>
</div>

Alert Pattern (Best Practice)

<div class="alert alert-success">
  <svg class="alert-icon" role="img" aria-label="Success">...</svg>
  <div class="stack-tight">
    <strong>Success!</strong>
    <p>Your file has been uploaded successfully.</p>
  </div>
</div>

Card Pattern (Best Practice)

<div class="feature-card">
  <svg class="feature-card-icon">...</svg>
  <h3 class="t-heading-sm">Feature Name</h3>
  <p class="t-body-sm text-muted">Brief description of this feature.</p>
</div>
