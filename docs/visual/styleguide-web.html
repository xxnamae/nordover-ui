What is a design system?

A design system is a collection of reusable components, patterns, and tokens that serve as the single source of truth for building digital products. It ensures consistency, accelerates development, and improves collaboration between design and engineering.

How do I customize components?

Override CSS variables (colors, spacing, fonts) using @layer brand rules in your CSS. Variables are documented in the tokens reference. All tokens are semantic and support both light and dark modes.

Is the framework accessible?

Yes. Nordover is built with WCAG AA compliance as a baseline. All interactive components include keyboard navigation, focus management, and screen reader support through semantic HTML and ARIA attributes.

Can I use this with my tech stack?

Yes. Nordover is pure CSS and HTML with no framework dependencies. Use it with React, Vue, Svelte, or vanilla JavaScript. The CSS is modular and can be imported partially.

What browsers are supported?

Modern browsers: Chrome 99+, Firefox 97+, Safari 15.4+, Edge 99+. CSS @layer and :has() required.

Can I customize colors?

Yes, override CSS variables in a @layer brand block. Change accent, adjust neutral hue, customize semantic colors.

Is dark mode supported?

Yes, via :root:has(#dark:checked) selector. All tokens have light/dark variants. Toggle built-in to styleguides.

Responsive Design

All components are mobile-first. Nordover uses clamp() for fluid sizing and container queries for layout adaptation.

Mobile-First Approach

Design for mobile first, then enhance for larger screens:

Default: Mobile layout (320px+)

@media (min-width: 48rem): Tablet/desktop enhancements

Note: No max-width limits — responsive to 1600px+ screens

Fluid Typography with clamp()

Type scales automatically scale between min and max values:

  • .t-heading-lg: 32px → 64px
  • .t-body: 16px base, responsive line-height

Accessibility (WCAG AA)

Nordover is built for accessibility. All interactive components include keyboard navigation, focus indicators, and semantic HTML.

Keyboard Navigation

Buttons: Tab to focus, Space/Enter to activate

Form Inputs: Tab to focus, Shift+Tab to reverse, arrows in selects

Modal Dialogs: Escape to close, focus trap inside modal

Accordion: Tab to focus summary, Space/Enter to toggle

Date Picker: Tab/Shift+Tab, Arrow keys for dates, Enter to select

Focus Indicators

All interactive elements have visible focus states (blue border outline + glow):

Semantic HTML

• Use semantic elements: <button>, <a>, <input>, <select>, <label>

• Always pair inputs with <label> for screen readers

• Use <h1>-<h6> for headings (not divs with styling)

• Use <nav>, <main>, <article>, <section> for landmarks

• Mark up lists with <ul>, <ol>, <li>

Accessibility Requirements

Focus visible: All interactive elements must show :focus-visible indicator (outline or background change). 3:1 contrast minimum.

Disabled appearance: Must be visually distinct - reduce opacity, change color, or add strikethrough. Not color alone.

Error states: Must be indicated by more than color. Include icon, text message, or border style change.

Motion: All animations respect prefers-reduced-motion. @media (prefers-reduced-motion: reduce) disables animations.
