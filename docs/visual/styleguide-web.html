How does this work?

The component uses semantic <details> and <summary> elements, which are fully accessible and keyboard-navigable by default.

Can I customize it?

Yes, override CSS variables (colors, spacing, fonts) via @layer brand block in your CSS.

Is it accessible?

Fully WCAG AA compliant. Keyboard navigation, focus management, and screen reader support built-in.

FAQ Section

Frequently asked questions. Uses accordion pattern with details/summary elements.

What browsers are supported?

Modern browsers: Chrome 99+, Firefox 97+, Safari 15.4+, Edge 99+. CSS @layer and :has() required.

Can I customize colors?

Yes, override CSS variables in a @layer brand block. Change accent, adjust neutral hue, customize semantic colors.

Is dark mode supported?

Yes, via :root:has(#dark:checked) selector. All tokens have light/dark variants. Toggle built-in to styleguides.
