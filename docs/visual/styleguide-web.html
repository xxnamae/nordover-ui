Motion Principles
✓ Always use duration tokens (--duration-*), never hardcoded values
✓ Respect prefers-reduced-motion via CSS @media query
✓ Use easing functions for natural movement (ease-out, cubic-bezier)
✓ Keep animations under 400ms for responsiveness
Easing Functions (Timing)

ease-out: Fast start, slow end — For entrances and positive actions

ease-in: Slow start, fast end — For exits (less common)

ease-in-out: Slow at both ends — For continuous movements

cubic-bezier(0.34, 1.56, 0.64, 1): Bouncy spring effect

Avoid: Linear and ease-in for entrances (feels mechanical)

CSS Implementation Guide

Best Practices Summary
  • ✓ All animations use CSS (not JavaScript when possible)
  • ✓ Animations follow motion tokens (--duration-*)
  • ✓ Easing is natural (ease-out for entrances)
  • ✓ Duration 150-400ms (never longer than 400ms)
  • ✓ prefers-reduced-motion always respected
  • ✓ Loading spinners have .icon-spin class
  • ✓ No animation on pure decorative elements

What is a design system?

A design system is a collection of reusable components, patterns, and tokens that serve as the single source of truth for building digital products. It ensures consistency, accelerates development, and improves collaboration between design and engineering.

How do I customize components?

Override CSS variables (colors, spacing, fonts) using @layer brand rules in your CSS. Variables are documented in the tokens reference. All tokens are semantic and support both light and dark modes.

Is the framework accessible?

Yes. Nordover is built with WCAG AA compliance as a baseline. All interactive components include keyboard navigation, focus management, and screen reader support through semantic HTML and ARIA attributes.

Can I use this with my tech stack?

Yes. Nordover is pure CSS and HTML with no framework dependencies. Use it with React, Vue, Svelte, or vanilla JavaScript. The CSS is modular and can be imported partially.

Works on all modern browsers (Chrome, Firefox, Safari, Edge). CSS Grid and Flexbox required. Progressive enhancement for older browsers—components degrade gracefully.

  • ✓ Page landmark regions announced: main, nav, sidebar, footer
  • ✓ Heading hierarchy is logical (no skips: h1 → h2 → h3)
  • ✓ Form labels are programmatically associated with inputs
  • ✓ Button text is descriptive (not "Click here")
  • ✓ Images have alt text (or aria-hidden if decorative)
  • ✓ Tables have <caption> or aria-label
  • ✓ Error messages are announced to screen readers
  • ✓ Dynamic content uses aria-live="polite"
