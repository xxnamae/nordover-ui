Responsive Design

Mobile-first framework. Test on real devices. Nordover adapts from 320px to 1600px+ without changes to HTML.

Key Principles
  • Mobile-first: Start with mobile layout
  • Fluid sizing: Use clamp() not breakpoints
  • Flexible grids: Use .grid-auto not rigid layouts
  • Touch-friendly: Min 44px tap targets on mobile
  • Test real devices: Not just browser DevTools
Breakpoint

Primary breakpoint: 48rem (768px) for tablet/desktop enhancements

For App Developers

All components work on mobile, tablet, desktop. Use responsive utilities and layout primitives. No additional CSS needed.

Accessibility

WCAG AA compliant. All components include keyboard navigation, focus indicators, and semantic markup.

Quick Checklist
  • ✓ All buttons keyboard navigable (Tab/Space/Enter)
  • ✓ Focus indicators visible on all interactive elements
  • ✓ Form labels associated with inputs
  • ✓ Color contrast 4.5:1 (WCAG AA)
  • ✓ Dark mode supported with no contrast loss
  • ✓ Semantic HTML used throughout
  • ✓ Screen reader tested (NVDA, VoiceOver)
Testing Tools

Recommended tools: axe DevTools, WebAIM Contrast Checker, NVDA (Windows), VoiceOver (Mac), Lighthouse

ARIA Implementation

aria-label — Icon buttons, close buttons

aria-expanded — Menu toggles (true|false)

aria-current — Active nav item (page)

aria-sort — Sortable table headers

aria-live="polite" — Alert messages

role="alert" — Important notifications

Keyboard Navigation (App-specific)
  • Tab: Move forward through interactive elements
  • Shift+Tab: Move backward
  • Space/Enter: Activate buttons, checkboxes
  • Escape: Close dialogs, modals, dropdowns
  • Arrow keys: Navigate lists, menus, date pickers
  • Enter: Select option in dropdown/date picker

Example — FAQ
How do I use the framework?

Import the CSS files (tokens and components), then use class names in your HTML. All components are self-contained and don't require JavaScript.

Can I customize colors?

Yes. Define CSS variables in @layer brand rules to override defaults. All tokens are semantic and will adapt to light/dark mode.

Is it mobile-friendly?

Yes. All components use mobile-first responsive design with clamp() for fluid sizing. Test on real devices for best results.
