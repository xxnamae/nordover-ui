🚀 Optimization Strategies & Recommendations

1. Critical CSS Extraction (Save 30-50ms FCP)

Extract 2-3 KB of critical CSS for inline delivery:

<style> /* Inline critical CSS (~2KB) */ @layer tokens, reset, primitives; /* Includes: font-face, box-sizing, typography, layout utilities */ </style> <link rel="stylesheet" href="components-web.css" async>
📈 Impact: Reduces FCP by ~30ms. Recommended for above-fold content.

2. Preload Web Fonts (Reduce LCP Impact)

<link rel="preload" href="/fonts/inter.woff2" as="font" type="font/woff2" crossorigin> <link rel="preload" href="/fonts/inter-tight.woff2" as="font" type="font/woff2" crossorigin>
📈 Impact: Fonts load 200-400ms earlier. Reduces LCP by 200-300ms.

3. Lazy Load Non-Critical Components

<!-- Load base styles --> <link rel="stylesheet" href="tokens-web.css"> <link rel="stylesheet" href="components-base.css"> <!-- Prefetch advanced components --> <link rel="prefetch" href="components-advanced.css">
📈 Impact: Non-critical components load after page is interactive. No impact on FCP/LCP.

4. Tree-Shake Utility Classes (Save 2-3KB)

Remove unused utility layer in production if app doesn't use .stack, .cluster, etc.

📈 Impact: Saves ~2-3KB (15% of gzipped size). Use for production builds only.

5. CSS Caching Strategy (1-Year TTL)

Cache-Control: public, max-age=31536000

Implementation: Use content-hash versioning in filename:

tokens-web-3.0.0.css components-web-3.0.0.css
📈 Impact: Repeat visits have zero CSS download. Huge savings for returning users.

6. Avoid These Performance Killers

Anti-Pattern Why It's Slow Alternative
width: 100% → 80% transition Triggers reflow (expensive) transform: scaleX(0.8)
background-color animation Triggers repaint on every frame opacity: 0.8; color-mix(...)
box-shadow animation Complex shadow calculations Use opacity on shadow element
Deep CSS selectors (6+ levels) Slow selector matching Keep selectors flat (2-3 levels max)
Animating on scroll Triggers layout recalculation Use IntersectionObserver API

Must-Do (High Impact)

  • Deliver CSS gzipped (18% of original size)
  • Preload web fonts (reduce LCP 200-300ms)
  • Cache CSS with max-age=31536000 + versioning
  • Test animations with DevTools (60 FPS check)
  • Verify CLS = 0 (no layout shift)

Should-Do (Medium Impact)

  • Extract critical CSS (2-3 KB) for inline delivery
  • Lazy-load non-critical components
  • Remove utility layer if not used (save 2-3 KB)
  • Use Brotli compression (10-15% better than gzip)

Nice-to-Do (Low Impact)

  • Analyze selector specificity (keep flat)
