1. Quick Start (5 min)

Nordover er en ren CSS-framework. Inkluder bare CSS-filene:

<!DOCTYPE html> <html> <head> <link rel="stylesheet" href="path/to/tokens-web.css"> <link rel="stylesheet" href="path/to/components-web.css"> </head> <body> <button class="btn btn-primary">Klikk meg</button> <input class="form-input" type="text" placeholder="Skriv her"> </body> </html>

Det er det! Du har nå tilgang til alle 40+ komponenter. Se styleguides for alle eksempler.

2. Token System (10 min)

Nordover bruker CSS-variabler (custom properties) for farger, spacing, typography, osv.

Farger

Gråskala: --gray-50 til --gray-950

Semantiske: --color-accent, --color-success, --color-error, --color-warning

Spacing

--space-1 (8px) til --space-8 (64px)

Typography

--text-sm, --text-base, --text-lg, --text-2xl, osv.

Alle variabler er definert i tokens-web.css. Du kan overstyre dem i ditt CSS:

:root { --color-accent: #ff0000; /* Rød istedet for svart */ --space-4: 20px; /* Større spacing */ }

3. CSS Klasse-konvensjoner (5 min)

Komponenter: .btn, .btn-primary, .btn-sm

Format: komponent-variant-size

Eksempler:

  • .btn .btn-primary .btn-sm = liten primær knapp
  • .form-input = tekstinput
  • .badge .badge-success = grønt badge
  • .btn .btn-primary .btn-elevated = knapp med skygge

Modifikators: Kombiner multiple klasser for ønsket resultat.

4. Dark Mode (3 min)

Nordover har innebygd dark mode via CSS custom properties.

Toggling settes opp med checkbox-input:

<input type="checkbox" id="dark" class="sr-only"> <label for="dark" style="cursor: pointer;">🌙</label>

CSS automatically swapper farger basert på checkbox-state:

:root:has(#dark:checked) { --color-bg: #1a1a1a; --color-fg: #ffffff; /* og mer... */ }

Theme persistes automatisk til localStorage. Test: Klikk på 🌙 øverst på siden!

5. Responsive Design (5 min)

Mobile-first: CSS is optimized for small screens by default.

Breakpoints:

  • 36rem (576px) = mobilcompressed
  • 48rem (768px) = tablet
  • 60rem (960px) = desktop

Responsive utilities:

.btn { padding: var(--space-3) var(--space-5); } @media (max-width: 48rem) { .btn { padding: var(--space-2) var(--space-4); } /* Mindre padding på mobil */ }

Touch targets: Minimum 44px × 44px for alle interactive elements.

6. Accessibility WCAG AA (5 min)

Nordover har WCAG AA compliance innebygd:

  • Keyboard navigation: Tab gjennom alle elementer, Enter/Space for aktivering, Escape for lukkinger
  • Focus states: 3px ring rundt fokusert element
  • Color contrast: Alle farger oppfyller minimum 4.5:1 for normal text
  • Dark mode: Samme kontrast i dark som light mode
  • Motion: Respektes prefers-reduced-motion system setting
  • Semantic HTML: Alle form inputs har labels, buttons har aria-labels, osv.

Testing: Bruk WAVE browser extension for accessibility audit.

7. Troubleshooting

Farger ser feil ut

Sjekk at du har linket begge CSS-filer. Hvis dark mode er på, kan det se annerledes ut. Klikk 🌙 for light mode.

Komponenter ser merkelige ut

Verifiser at du bruker korrekt CSS-klassifiering. Bruk styleguiden som referanse.

Spacing virker rart

Husk at Nordover bruker gap, margin, og padding fra token-systemet. Overstyr ikke disse med inline styles.

Dark mode persisteres ikke

localStorage må være aktivert i nettleseren din. Sjekk browserinstillinger.

For flere eksempler, se styleguide-web.html og styleguide-app.html

  • .form-input = text input
  • .badge .badge-success = green badge
  • .btn .btn-primary .btn-elevated = button with shadow

Modifiers: Combine multiple classes to achieve your desired result.

7. Troubleshooting

Colors look wrong

Make sure both CSS files are linked. If dark mode is on, colors will look different. Click 🌙 for light mode.

Components look weird

Verify you're using the correct CSS class names. Use the styleguide as reference.

Spacing seems odd

Remember that Nordover uses token values for gap, margin, and padding. Don't override with inline styles.

Dark mode doesn't persist

LocalStorage must be enabled in your browser. Check your browser settings.
